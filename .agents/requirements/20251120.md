## 页面

- [ / ] 仪表盘（首页）

仪表盘的基本功能已经大体设计完成，主要的任务是填充下方的卡片。卡片应该以简约但突出色的设计风格为主，尽量采用艺术字等。

Title 下方使用 grid columns 3 的卡片展示主要分栏：服务器情况、名下数据、申请流程。 

再下方是根据服务器数据生成的卡片。

首页的时候 Header 中部有描述文字背景也有图片。如果点击图片可以进入预览模式。

- [ /profile ] 玩家档案

类似 GitHub 的布局风格。左侧用户最近登录 IP 的小地图，下方用户信息（Minecraft ID、Minecraft Nick、系统头像和玩家头颅、PIIC 和其条形码、档案信息概要，比如地址性别地区最近登录、是否绑定用户）；下方游戏内的历史操作记录。

右侧各种卡片展示玩家名下数据（公司、铁路）、所属行政区信息、服务器数据 AuthMe 数据 + 权限组数据 + Authme 密码重置、强制登陆、统计信息（飞了多远、上线多久）、自助修改权限组；炸服了申请强制重启。【调整公开信息，不过得放设置里面】

- [ /rank ] 排行榜
特定类别的好玩的统计数据排行。上线排行榜。

- [ /databases ] 服务器公开数据库

用于查询管理员数据、玩家和用户数据、服务端运行情况、通过玩家凭证查询玩家档案验证报告。

- [ /administration ] 行政区划（/administration/division）和管理机构公示

- [ /company ] 个体户和公司数据查询、申报、管理页面

- [ /railway ] MTR、机械动力铁路数据查询、申报、管理页面

- [ /gallery ] 图集
